# End-to-End Predict-and-Optimize (端到端决策学习框架)

本项目实现了一个基于“预测-优化”流水线的端到端库存决策学习框架。通过神经网络进行需求预测，ABCA启发式算法进行运筹求解，并利用 LightGBM 代理模型（Surrogate Model）将不可导的业务成本转换为可传导的梯度，直接驱动预测网络以“最小化最终业务成本”为目标进行优化。

## 🚀 启动训练

可以通过命令行运行端到端训练脚本，支持以下参数：

```bash
python project/main.py [参数...]
```

### 命令行参数说明

| 参数             |  类型   | 默认值 | 说明                                                                                                                                                                                        |
| :--------------- | :-----: | :----: | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--epochs`       |  `int`  |  `5`   | 训练的 Epoch 数量。建议先使用 `1` 或 `2` 进行测试。                                                                                                                                         |
| `--report_to`    |  `str`  | `none` | 是否上报指标到特定平台。设为 `"wandb"` 可启用 Weights & Biases 画图。                                                                                                                       |
| `--exp_name`     |  `str`  | `8008` | 实验/项目名称。当 `report_to=wandb` 时，作为 wandb project name。                                                                                                                           |
| `--penalty_coef` | `float` | `10.0` | **缺货声誉惩罚系数（核心业务参数）**。缺货不仅损失当前商品的毛利，还会导致客户流失。此系数定义了缺货导致的额外惩罚是“单件商品售价的多少倍”。系数过低（如 0.0）会导致模型摆烂预测 0 不进货。 |

**示例命令**：

```bash
# 运行 3 个 Epoch，开启 wandb 记录，并将缺货惩罚倍数设为 15 倍
python project/main.py --epochs 3 --report_to wandb --penalty_coef 15.0
```

## 📊 核心监控指标 (Metrics) 解析

在训练过程中（特别是使用 wandb 时），`loop.py` 当前会记录以下字段：
`total_loss`、`cost_loss`、`pred_loss`、`mse_loss`、`mean_true_cost`、`mean_y_pred`、`mean_abs_grad`。

为了写报告和看训练曲线更清楚，建议把它们分成“主要指标”和“次要指标”。

### 1. 主要指标（建议正文重点展示）

#### `mean_true_cost` (真实业务成本均值)

- **定位**：**最主要**，是整个系统的北极星指标。
- **含义**：求解器根据当前预测做出订货决策后，在真实环境中与真实需求碰撞所产生的实际业务成本。
- **目标**：**越低越好**。它直接决定端到端方案在业务上的最终表现。

#### `cost_loss` (代理业务成本损失)

- **定位**：**主要**，是业务目标在训练阶段的可导近似。
- **含义**：代理模型基于当前预测值和成本上下文估计出的成本，用于给预测网络提供可反向传播的业务梯度。
- **目标**：**越低越好**。它通常应与 `mean_true_cost` 的趋势大体一致；如果两者明显脱节，说明代理模型拟合可能失效。

#### `total_loss` (总损失)

- **定位**：**主要**，是优化器真正最小化的训练目标。
- **含义**：由 `cost_loss` 和 `pred_loss` 组合得到。`weighted_sum` 下是直接加权和，`balanced_sum` 下是归一化后的加权和。
- **目标**：**越低越好**。它主要用于判断训练是否稳定收敛，但因为带有策略和权重，不宜单独作为业务结论。

#### `pred_loss` (预测误差损失)

- **定位**：**主要**，是约束预测质量的辅助目标。
- **含义**：在 `log1p` 空间上计算的 Huber Loss，用来减少长尾需求和极端值对训练的冲击。
- **目标**：**越低越好**。它帮助模型在优化业务成本的同时，不至于完全偏离真实需求。

### 2. 次要指标（建议作为补充分析或训练诊断）

#### `mse_loss` (原始需求空间的 MSE)

- **定位**：**次要**，用于和传统预测任务对照。
- **含义**：在原始需求空间计算的均方误差，不直接参与当前训练目标。
- **目标**：**越低越好**。适合放在补充实验里说明预测误差水平。

#### `scaled_cost_loss` / `scaled_pred_loss` (缩放后的损失分量)

- **定位**：**次要**，用于解释 `total_loss` 的内部构成。
- **含义**：在 `weighted_sum` 或 `balanced_sum` 策略下，两部分 loss 进入总损失前的实际贡献值。
- **目标**：没有单独的业务目标，主要用来看当前训练更偏向优化哪一部分。

#### `ema_total_loss` / `ema_cost_loss` / `ema_pred_loss` (指数滑动平均)

- **定位**：**次要**，用于平滑训练曲线。
- **含义**：对应 loss 的指数滑动平均版本，减少 batch 级噪声，便于观察长期趋势。
- **目标**：通常希望平稳下降，但它们只是展示手段，不是独立优化目标。

#### `mean_abs_grad` (代理梯度绝对值均值)

- **定位**：**次要但很有用**，是收敛诊断指标。
- **含义**：代理模型传回的伪梯度绝对值均值，反映“预测值再变化一点，成本还会不会明显变化”。
- **目标**：一般希望逐渐变小并趋于稳定。若长期很大，说明系统还在剧烈调整；若突然接近 0 且业务成本仍很高，可能掉进了平坦区。

#### `mean_y_pred` (预测需求均值)

- **定位**：**次要**，用于检查预测输出是否异常。
- **含义**：当前 batch 内模型预测需求的平均值。
- **目标**：没有固定的越高越好或越低越好标准，但不应长期塌缩到 0，也不应出现无意义的大幅震荡。

### 3. 报告撰写建议

- **结果表主指标**：优先写 `mean_true_cost`。
- **训练过程主曲线**：建议展示 `total_loss`、`cost_loss`、`pred_loss`。
- **补充诊断图**：可选加入 `mean_abs_grad`、`mse_loss`、`mean_y_pred`。
- **机制解释图**：如果需要说明 `balanced_sum` 或 `weighted_sum` 的效果，再展示 `scaled_cost_loss`、`scaled_pred_loss` 和各类 `ema_*`。
